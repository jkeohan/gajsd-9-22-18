JS Frameworks & Libraries

JS Frameworks & Libraries

Library ⇒

• Set of predfined functions that your code calls

• Each call performs work and returns a result (and control) to your code

• Specific, well-defined operations

• Examples: jQuery and LoDash

JS Frameworks & Libraries

Framework ⇒

• Opinionated architecture for building software

• Control-flow exists, you fill in with your code

• Calls your code; is always in control

• Examples Angular and Ember

JS Frameworks & Libraries

• The primary difference (source):

• You call library

• Framework calls you

Git and Github

git: What is it?

• git is a version control program

• Watches files and records any changes

• Allows developers to save the current status (version) of a project

• Helps multiple developers coordinate on the same project

• git ≠ GitHub

Git: That Learning Curve
GitHub: What Is It?

• GitHub is a platform that makes it easy to work with git

• Like Dropbox, but for code

• Stores a history of files and their changes

• Hosts files in the cloud; easy to share with other devs

• Also serves as a social network (of sorts) for developers collaborating on projects

GitHub: Why Do Developers Use It?

• Tracks changes; see who worked on (or, broke) what

• Stores history; can easily revert and fix mistakes

• Allows multiple developers to work on the same project

• Allows for line-by-line feedback

GitHub: some vocab words

1. Repository

2. Clone

3. Commit

4. Push

GitHub: Repository/Repo

• Basic element of GitHub

• Contains all of a project's files (all the code)

• One or more users can contribute to a single repository

• Repositories are either public or private

• By EOC, you will each create your own repo!

GitHub: Clone

• Copies/clones a remote repo to your machine

• This copy/clone is called a local repo

• Changes to the local repo will not affect the remote

• Multiple users can clone the same repo!

GitHub: Commit

• A snapshot of changes to a repo

• Think of it as "saving" your changes

• Commits do not change the original (remote) repo

• Contains a message describing the changes made

GitHub: Push

• "Pushes" your commits (saved changes) to a remote repository

• Pushing does update the original (remote) repo

• Allows other developers to see your changes and "pull them down" to their own local repos

GitHub: Committing and Pushing Your Changes

1) Use git add -A to add/stage files for a commit

2) Run git status - untracked files should be tracked!

3) To commit: git commit -m "Commit msg here!"

4) To push your local changes to remote: git push origin master

5) Visit your repo on http://github.com to ensure changes have been made
