Functions

Functions

• Blocks of code that execute (or run) line-by-line

• Reusable statements that can be called anywhere in the program

• Often take an input, perform logic on that input, and return ouput

• Allow the developer to segment large applications into smaller pieces

• Think of them as miniature programs

Functions

Functions must be declared (defined) before use:

1) Function Declarations:


	function speak (words) {
	  console.log(words)
	}

2) Function Expressions:


	var speak = function (words) {
	  console.log(words)
	}
Functions

• Function declarations can be used before they are defined


	speak('hello, world!');

	function speak (words) {
	  console.log(words);
	}

	// 'hello, world!'

• Function expressions must be defined before they are called


	speak('hello, world!');

	var speak = function (words) {
	  console.log(words);
	}

	// TypeError: undefined is not a function
Functions

A function declaration has the following syntax:

• Name: used to invoke or call the function

• Parameters: options to change the function's input

• Body: code that runs when the function is called


	function speak (words) {
	  console.log(words);
	}

	// multiple parameters
	function speak (words, repeat) {
	  for (var i = 1; i < repeat; i++) {
	    console.log(words)
	  }
	}
Functions

• Remember: you must call functions to use them!

• Calling a function will execute the function's body

• To call a function, follow the function's name with ()


	// function declaration
	function helloWorld () {
	  console.log('hello, world!');
	}

	// will not call the function; returns the definition
	helloWorld

	// use () to call a function
	helloWorld('hello')

Parameters

Parameters

• If a function performed the same operation every time, it wouldn't make a dynamic codebase

• We would have to write a new function for small variations

• For example, if we wanted to write functions that greeted the entire cast of Point Break:


  // Bad idea...
  function helloKeanu () {
    console.log('Hello, Keanu');
  }

  function helloGary () {
    console.log('Hello, Gary');
  }
Parameters

• Function parameters remedy this problem by allowing us to call the same function with different values:


	function sayHello (name) {
	  console.log('Hello, ' + name);
	}

	sayHello('Keanu');
	sayHello('Gary');

	// 'Hello, Keanu'
	// 'Hello, Gary'
Parameters

• Data inputs that are "passed" into a function

• Defined as a comma-separated list:


	function add (x, y) {
	  return x + y;
	}

	function addThree (x, y, z) {
	  return x + y + z;
	}

	add(2, 2);
	// 4

	addThree(2, 4, 6)
	// 12

The return Statment

The return statement

• The return statement allows us to treat functions as values

• Ends the function execution and returns a value from the function


	function sum (x, y) {
  	  return x + y;
	}

	var z = sum(3, 4);
	// 7

• Optional - if absent the function returns undefined

The return statement

Passing return values to another function:


	function sum (x, y) {
  	  return x + y;
	}

	function dbl (z) {
	  return z * 2;
	}

	var num = sum(3, 4);
	// 7

	var dblNum = dbl(num)
	// 14

	// We can also write:
	var otherNum = dbl(sum(3, 4));
	// 14
The return statement

• The return statement STOPS function execution:


  function speak (words) {
    return words;

    // The following statements will not run:
    var x = 1;
    var y = 2;
    console.log(x + y)
  }

• Useful when applied to conditionals:


  function authenticate (user) {
    if (user !== 'authenticated') {
      return authenticate(user);
    }

    return allowEntry(user);
  }

Scope

Scope

• Scope is the set of variables, objects, and functions your code has access to

• Two levels of scope:

⇒ Global

⇒ Local

• In JS, scope is created by functions and blocks

• What will the following code return?


	var num = 10;

	function speak () {
	  var num = 100;
	  return num;
	}

	speak()
Scope: Global Scope

• Variables, objects, and functions defined OUTSIDE of a function

• Public; accessible from all other scopes

• Any script or function may access:


	// global scope
	var myFavActor = 'Keanu Reeves';

	function alertFavActor () {
	  // local scope, but has access to global scope
	  console.log(myFavActor);

	  // automatically global
	  favFilm = 'Point Break';
	}

• If you omit var, the variable becomes global

Scope: Local Scope

• Scope created by a block or function

• Variables created inside of functions cannot be accessed outside the function


	var a = "this is the global scope";

	function myFunction() {
  	  var b = "this variable is defined in the local scope";
	}

	myFunction();

	console.log(b);
	// ReferenceError: b is not defined

• Typically one global and many local

Scope: Creation

Function Scope:


	function speak(words) {
	  // scope created by the speak function
	  var message = words + "!";
	}

Block Scope:

• Scope created by loops and conditionals

• ES6 feature!


	if (true) {
	  let x = 2;
	}

	console.log(x)
	// ReferenceError: x is not defined
Scope: Lexical Scope and Closures

• Inner functions have access to the scope of the outer functions


  // global scope

  function speak (words) {
    // speak scope
    var myFavoriteActor = 'Keanu Reeves'

    function sayFavActor () {
  	  // sayFaveActor scope - has access to speak's scope
	  console.log(myFavoriteActor)
    }
  }
