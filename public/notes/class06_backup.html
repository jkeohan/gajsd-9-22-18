Objects: Introduction

• Objects are collections of related data and functions


  var person = {
    name: 'Chandler',
    age: 29,
    favorites: ['JavaScript', 'beer', 'beach'],
    speak: function () {
      return 'Do your JavaScript homework!';
    },
  }

• Object values (ex: name) are called properties

• Object functions (ex: speak) are called methods

• Syntax follows a comma-separated, key/value pattern

Objects: Introduction

Objects: Read/Write Properties

• Use dot-notation or bracket notation to read/write propreties


  var person = {
    name: 'Chandler'
  };

  // dot-notation
  person.job = 'frontend engineer';

  person.job
  // "frontend engineer"

  // bracket notation (useful if a property has a special character)
  person['bad-habits'] = 'beer';

  person['bad-habits'];
  // "beer"

Objects: Property Iteration

• Use for...in loop to iterate through an object's enumerable properties

• NOTE: unlike arrays, objects do not maintain property order


    var person = {
      name: 'Chandler',
      age: 29,
    };

    for (var key in person) {
      console.log(key);
      console.log(person[key]);
    }

    // name
    // "Chandler"
    // age
    // 29
Objects: Property Iteration

• Can also use Object.keys() for property iteration:


  var person = {
    name: 'Chandler',
    age: 29,
  };

  Object.keys(person);
  // ["name", "age"]

  Object.keys(person).forEach(function (key) {
    var value = person[key];
    console.log(value)
  })

  // "Chandler"
  // 29

• NOTE: Object.values() is still in development

Object Constructors

Object Constructors

• Factories used to create many of the same type of object


  // Object constructor
  var Person = function(name) {
    this.name = name;
  };

  // An instance of the Person constructor
  var chandler = new Person('Chandler');
  var keanu = new Person('Keanu');

• By convention, use upper-cased names when defining constructors

• Invoke the constructor with the new keyword

• NOTE: JS is a prototype-based language; there are no classes!

Object Constructors

• Object constructors are used to set an object's properties:


  var Person = function(name, age) {
    this.name = name;
    this.age = age;
  }

  var chandler = new Person('Chandler', 29)

• More reusable than creation via literal:


  var chandler = {
    name: 'Chandler',
    age: 29,
  }
Object Constructors: this

• this is used to set a function's context

• Inside the object constructor, this refers to the current object instance

• Must use this when referring to the object using methods:


  var Person = function (name) {
    this.name = name;
    this.sayName = function () {
      console.log(this.name);
    }
  }

  var chandler = new Person('chandler');
  chandler.sayName();
  // "chandler"

JavaScript Object Notation (JSON)

JSON: What is it?

• JSON: JavaScript Object Notation

• JSON !== JavaScript

• Text-based data format that is based on JS

• JSON is language-independent; Ruby, Python, etc. can all parse/generate JSON

• Successor to XML (ew)

• Easy for humans to read; easy for machines to parse

• Specified by Douglas Crockford

JSON: What's it used for?

• JSON is used as a data-exchange format between programs

• Ex: sending data from the frontend (JavaScript), to a Python-powered backend, and vice-versa

• Has strict conventions (unlike JS):

⇒ Property names must be double-quoted strings: "name" vs name

⇒ All strings must be double-quoted: "name" vs 'name'

⇒ No in-line comments

⇒ Trailing commas are forbidden

JSON: JavaScript's JSON Object

• Like the Math object, JavaScript has a lib for working with JSON: JSON

• JSON.stringify() ⇒ Transforms JavaScript to JSON


  var jsonObject = JSON.stringify({ name: 'chandler' });
  // "{"name":"chandler"}"

  typeof jsonObject
  // string

• JSON.parse() ⇒ Transforms JSON to JavaScript


  var javaScriptObject = JSON.parse(jsonObject);
  // { name: "chandler" }

  typeof javaScriptObject
  // "object"
